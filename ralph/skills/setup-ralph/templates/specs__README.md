# specs/

One spec file per feature or module. Each spec should contain:

- **Goal** — one sentence on what this feature is for.
- **Requirements** — bullet list of must-haves.
- **Non-goals** — what this explicitly does *not* cover.
- **Acceptance tests** — concrete checks that prove the feature works.

Specs are the loop's target. The agent reads them; do not let it invent product requirements from vibes.

Form specs by **conversation** with the agent before the loop starts running. Then ask the agent to write them out, one per file.
